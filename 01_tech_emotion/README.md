# Paper 1: Technological Feasibility & Emotional Transmission in VR

## Project Focus
This paper synthesizes technological trends, nonverbal communication channels, and interpersonal emotions. It establishes a Business Intelligence (BI) forecasting model to evaluate which professional meetings are "VR-ready" today versus the near future based on emotional transmission capabilities.

## Local Data Dependencies
* `Trends.csv`, `Nonverbal communication.csv`, `Nonverbal communication + Trend.csv`, `interpersonal emotions.csv`.
* External data inputs: Patent filings, physiological limits, TRL frameworks.

## Sprints / Work Packages
### Sprint 1: Multi-Source Data Ingestion
* Ingest and clean existing datasets.
* Integrate patent data and Technology Readiness Levels (TRL) to solidify expert estimations.

### Sprint 2: Theoretical Mapping
* Create a matrix connecting nonverbal communication channels to specific interpersonal emotions.

### Sprint 3: BI Forecasting Engine
* Develop a fault-tolerant forecasting model triangulating announcements, patents, and hardware specs to predict VR transmission capabilities.

### Sprint 4: Use Case & Longitudinal Setup
* Develop a scoring algorithm ranking the VR-readiness of specific meetings.
* Design the longitudinal BI framework (KPIs and data pipelines) to track real-world VR meeting usage over time.

### Sprint 5: Manuscript Drafting
* Write the scientific article (Intro, Methodology, Results, Discussion), explicitly framing the BI forecasting methodology.
